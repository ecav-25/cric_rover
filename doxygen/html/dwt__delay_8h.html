Detailed Description

Driver per ritardi ad alta precisione basati su periferica DWT.

Author
Gruppo 2

Questo modulo utilizza il registro CYCCNT (Cycle Counter) dell'unità Data Watchpoint and Trace (DWT) dei core ARM Cortex-M per implementare ritardi bloccanti con risoluzione al microsecondo.

◆ DWT_Delay_Init()

dwt_status_t DWT_Delay_Init ( void )

Abilita il conteggio dei cicli DWT->CYCCNT.

Deve essere chiamata una sola volta all'avvio del sistema (post clock config).

Returns
dwt_status_t DWT_OK se l'inizializzazione ha successo.

Abilita il conteggio dei cicli DWT->CYCCNT.

Abilita il bit TRCENA nel registro DEMCR del CoreDebug e il bit CYCCNTENA nel registro di controllo DWT. Esegue un test pratico per verificare che il registro CYCCNT stia effettivamente incrementando.

Returns
dwt_status_t DWT_OK se il contatore è attivo e funzionante, DWT_ERR_UNSUPPORTED se l'hardware non supporta DWT o l'abilitazione fallisce.

◆ DWT_Delay_ms()

dwt_status_t DWT_Delay_ms ( uint32_t ms)

Esegue un ritardo bloccante in millisecondi.

Utilizza il DWT invece di HAL_Delay per evitare dipendenze dal SysTick.

Parameters
msNumero di millisecondi di attesa.
Returns
dwt_status_t DWT_OK se il delay è stato eseguito.

Implementato come loop di chiamate a DWT_Delay_us(1000) per evitare overflow nel calcolo dei cicli per ritardi lunghi (> alcuni secondi a frequenze elevate).

Parameters
msNumero di millisecondi di ritardo.
Returns
dwt_status_t DWT_OK se eseguito, DWT_ERR_NOT_READY se non inizializzato.

◆ DWT_Delay_us()

dwt_status_t DWT_Delay_us ( uint32_t us)

Esegue un ritardo bloccante in microsecondi.

Parameters
usNumero di microsecondi di attesa.
Returns
dwt_status_t DWT_OK se il delay è stato eseguito, errore se DWT non init.

Calcola il numero di cicli necessari basandosi su SystemCoreClock. $$Cycles = \frac{SystemCoreClock \cdot \mu s}{1000000}$$

Parameters
usNumero di microsecondi di ritardo.
Returns
dwt_status_t DWT_OK se eseguito, DWT_ERR_NOT_READY se non inizializzato.
